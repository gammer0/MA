## 2026-05-06 (续2) — 加密密钥存储 + 安全注册流程重构

### 背景
解决了密钥管理中的两个核心问题：
1. **管理员不应当持有私钥** — 原来注册响应中返回 `private_key_pem`，管理员需要手动注入
2. **进程重启密钥丢失** — 原来私钥只在内存中，demoapp 重启后需要重新注册

### 新增文件
**`agent_sdk/key_store.py`** — 加密密钥存储模块
- `save_keys(keys)` — 用机器标识派生密钥 → XSalsa20-Poly1305 加密 → 写入 `~/.agent-secrets/keys.enc`
- `load_keys()` — 解密读取本地加密文件
- `delete_keys()` — 删除密钥文件
- 加密绑定机器标识（Windows SID / Linux machine-id / macOS UUID），文件拷贝无效

### 修改文件

| 文件 | 改动 |
|------|------|
| `agent_sdk/agent_registry.py` | 构造函数新增 `key_file`/`salt_file` 参数；启动时 `_restore_from_disk()` 自动恢复；`inject_keys()` 后自动 `_persist_to_disk()` 加密持久化 |
| `agent_sdk/__init__.py` | 导出 `AgentRegistry`、`save_keys`、`load_keys`、`delete_keys` |
| `feishu-demo-app/main.py` | `AgentRegistry` 传入密钥文件路径；去掉环境变量注入逻辑（由加密文件恢复替代） |

### 安全注册流程（修改后）
```
注册时:
  identity-service 推送私钥 → demoapp POST /admin/keys
  → AgentRegistry.inject_keys()
    → 创建 Agent 实例（内存）
    → 加密写入 ~/.agent-secrets/keys.enc（磁盘持久化）
  → 响应 200（不返回私钥，管理员不可见）

重启后:
  → AgentRegistry.__init__()
    → _restore_from_disk() 解密恢复
    → 创建 Agent 实例，无需重新注册

密钥变更:
  → inject_keys() 更新实例 + 自动更新加密文件
```
